기존 방식으로 테스트할땐 사이드이펙트등 문제가 있다.  

유닛 테스트용 모듈을 사용함으로써, 테스트할 모듈의 인터페이스만 동일하다면,   
같은 코드로 테스트를 수행할 수 있다.

TDD란? 테스트 주도 개발  
기존 :  
1. 코드를 짜고
2. 테스트를 만든다.  

TDD :  
1. 테스트를 만들고 (컴파일도 안되는)  
2. 컴파일을 되게해서 실행하는 테스트를 보고
3. 설계같은거 다 무시하고 성공하도록 어떻게든 바꾸고
4. **리팩토링한다.**

테스트를 위해 private 요소를 public으로 빼야할 것 같은 기분이 든다...?  
테스트하기 어려운 구조는 설계적으로 잘못가고있을 확률이 높다!!  


(단일 책임 원칙 - 클래스 하나는 하나의 책임을 가진다.)
